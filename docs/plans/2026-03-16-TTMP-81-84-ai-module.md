> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Синхронизация с Flow Universe («бой») — скрипт `backend/scripts/sync-issue-with-battle.mjs` (любые ключи задач).

**Goal:** Взять задачи TTMP-81 (EPIC), TTMP-82 (STORY), TTMP-83 (TASK), TTMP-84 (TASK) из Flow Universe, выполнить по правилам (план, декомпозиция, проверка), вернуть на бой статусы, комментарии и при необходимости описания.

**Architecture:** Иерархия 81 → 82 → 83/84. EPIC/STORY — контейнеры; код и верификация в 83 (POST /ai/estimate) и 84 (POST /ai/decompose). Pull/Push с Flow Universe через HTTP API (Bearer token).

**Tech Stack:** Node 20, Express, Prisma, React, Ant Design; скрипт синхронизации — ESM + fetch.

---

## Как пользоваться (кратко)

1. **Забрать с боя:** запустить бэкенд Flow Universe, взять access token (логин в UI → Local Storage `accessToken` или POST /api/auth/login). Затем:
   ```bash
   cd backend
   export TASKTIME_BASE_URL="http://localhost:3000"
   export TASKTIME_ACCESS_TOKEN="<твой accessToken>"
   node scripts/sync-issue-with-battle.mjs pull TTMP-81 TTMP-82 TTMP-83 TTMP-84 --set-in-progress
   ```
   Появится снимок `docs/plans/snapshot-TTMP-81-TTMP-82-TTMP-83-TTMP-84.json` и у всех четырёх задач в Flow Universe станет aiExecutionStatus = IN_PROGRESS.

2. **Выполнить задачи:** по плану ниже (Task 1–4) — проверка 83/84, при необходимости доработки.

3. **Вернуть на бой:** при необходимости поправить комментарии в `docs/plans/2026-03-16-TTMP-81-84-push-comments.json`, затем:
   ```bash
   cd backend
   export TASKTIME_ACCESS_TOKEN="<token>"
   node scripts/sync-issue-with-battle.mjs push TTMP-81 TTMP-82 TTMP-83 TTMP-84 --comments docs/plans/2026-03-16-TTMP-81-84-push-comments.json
   ```
   У всех четырёх задач выставится aiExecutionStatus = DONE и добавится комментарий с отчётом.

## Phase 0: Забрать с боя (Pull from Flow Universe)

Получить актуальные данные четырёх задач из Flow Universe и сохранить в репозиторий.

**Files:**
- Скрипт: `backend/scripts/sync-issue-with-battle.mjs`
- Create: `docs/plans/snapshot-TTMP-81-TTMP-82-TTMP-83-TTMP-84.json` (результат pull)

**Step 1: Убедиться, что бэкенд Flow Universe доступен и есть токен**

- Запустить бэкенд (`make backend` или аналог) или использовать URL стенда.
- Получить access token: залогиниться в UI (admin/manager), в DevTools → Application → Local Storage → `accessToken`, либо через `POST /api/auth/login` с телом `{ "email", "password" }` → в ответе `accessToken`.

**Step 2: Выполнить pull скриптом**

Скрипт `backend/scripts/sync-issue-with-battle.mjs` — один для любых ключей.

```bash
cd backend
export TASKTIME_BASE_URL="http://localhost:3000"   # или URL стенда
export TASKTIME_ACCESS_TOKEN="<accessToken>"
node scripts/sync-issue-with-battle.mjs pull TTMP-81 TTMP-82 TTMP-83 TTMP-84 --set-in-progress
```

- Скрипт выполняет `GET /api/issues/key/:key` для каждого ключа с заголовком `Authorization: Bearer <token>`.
- Результат сохраняет в `docs/plans/snapshot-TTMP-81-TTMP-82-TTMP-83-TTMP-84.json` (по ключу: id, title, description, status, aiExecutionStatus и т.д.).
- Флаг `--set-in-progress` выставляет для каждой задачи `PATCH /api/issues/:id/ai-status` → `IN_PROGRESS`.

**Критерий приёмки Phase 0:** Файл `docs/plans/snapshot-TTMP-81-TTMP-82-TTMP-83-TTMP-84.json` существует и содержит по ключу TTMP-81..84 актуальные title, description, id, aiExecutionStatus.
