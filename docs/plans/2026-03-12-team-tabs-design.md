# Team Tabs Design

## Goal

Добавить в левый сайдбар два новых раздела между `Projects` и `Sprints`: `Бизнес-функциональные команды` и `Потоковые команды`, чтобы показать будущую продуктовую модель команд через синтетические данные без изменения backend.

## Navigation

- Порядок пунктов в сайдбаре:
  - `Dashboard`
  - `Projects`
  - `Бизнес-функциональные команды`
  - `Потоковые команды`
  - `Sprints`
  - остальные существующие пункты без изменений
- Оба новых раздела являются top-level маршрутами.

## Data Strategy

- На первом шаге используем только фронтендовые синтетические данные.
- Backend, Prisma, DTO и API не меняются.
- Синтетика должна выглядеть как реалистичная заготовка для будущей продуктовой модели:
  - владельцы команд;
  - зоны ответственности;
  - фокус квартала;
  - synthetic KPI;
  - связанный backlog / активные спринты / риски.

## Business Functional Teams

- Новый маршрут: `/business-teams`.
- Формат страницы: полная ширина `tt-page` без aside.
- Основной паттерн: карточки, визуально близкие к проектам, но с продуктовым акцентом.
- Каждая карточка показывает:
  - название команды;
  - домен / продуктовую область;
  - owner и состав;
  - основной KPI;
  - active initiatives;
  - связанные спринты;
  - короткий synthetic backlog.
- В верхней части страницы показываем summary-метрики по количеству команд, инициатив и активных спринтов.

## Flow Teams

- Новый маршрут: `/flow-teams`.
- Формат страницы: полная ширина `tt-page` без aside.
- Основной паттерн: потоковые operational cards + таблица/список задач потока.
- Каждая карточка показывает:
  - название потока;
  - stage или value stream;
  - throughput / WIP / SLA synthetic metrics;
  - ближайший sprint bucket;
  - bottleneck;
  - несколько synthetic flow issues.
- Акцент на том, что потоковые команды работают с набором задач, которые позже можно собирать в спринты.

## UX Notes

- Страницы должны вписываться в текущую dark Linear-like тему.
- Используем существующие `tt-*` паттерны (`tt-page`, `tt-panel`, `tt-stats-grid`, `tt-table`) и добавляем только недостающие team-specific классы.
- На мобильной ширине карточки должны складываться в одну колонку.
- Не используем скрытую вложенную навигацию: оба раздела видны в sidebar напрямую.

## Testing

- Добавить e2e-сценарий, который проверяет:
  - наличие новых пунктов меню;
  - переход на обе страницы;
  - видимость ключевых заголовков и synthetic cards.
- После реализации прогнать релевантный e2e-тест, линтер и сборку фронтенда.
