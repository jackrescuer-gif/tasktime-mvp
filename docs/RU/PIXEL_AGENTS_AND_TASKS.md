# Интерфейс для ИИ-команды и визуализация (Pixel Agents)

**Важно:** Flow Universe — это система для **пользователей** (люди, задачи, учёт времени). Для постановки задач **ИИ-команде** и работы с их скиллами нужен **отдельный интерфейс**. Ниже — как это должно выглядеть и где что запускать.

---

## Разделение систем

| Система | Назначение |
|--------|------------|
| **Flow Universe** | Для людей: задачи пользователей, исполнители, учёт времени. Не используется для очереди задач ИИ-агентам. |
| **Интерфейс для ИИ-команды** | Отдельный интерфейс: постановка задач агентам, выбор роли/скилла, очередь задач для агентов. Агенты (в Cursor или VS Code) забирают задачи отсюда. |

---

## Как это будет выглядеть

### 1. Отдельный интерфейс постановки задач ИИ-команде

Отдельный экран или приложение (не Flow Universe), где вы:

- **Ставите задачу** — текст задачи, описание, приоритет.
- **Выбираете, кому** — роль или скилл (архитектор, разработчик, тестировщик, аналитик и т.д.), либо «любой свободный агент».
- **Видите очередь** — список задач для агентов: открытые, в работе, выполненные. Можно с телефона или из браузера.

Визуально это может быть:

- Отдельная страница (например `/ai-team` или отдельный хост): форма «Новая задача для ИИ», выпадающий список «Роль/скилл», список карточек задач с статусами.
- С телефона — тот же интерфейс, адаптированный под мобильный: большие кнопки, форма, список задач.

Данные для этого интерфейса хранятся отдельно от пользовательских задач Flow Universe (отдельная очередь/таблицы или отдельный сервис).

### 2. Где работают агенты: Cursor или VS Code

- **Всё из Cursor:**  
  Агенты запускаются в **Cursor** (встроенные агенты, чаты). Отдельный интерфейс постановки задач отдаёт очередь по API. Агент в Cursor по скиллу (например `tasktime-inbox` или будущий `ai-queue`) запрашивает «задачи для ИИ», берёт одну в работу, обновляет статус.  
  **VS Code не обязателен** — можно поднять весь цикл в Cursor: ставите задачи в отдельном интерфейсе, работаете с агентами в Cursor.

- **Визуализация Pixel Agents (пиксель-офис):**  
  [Pixel Agents](https://github.com/pablodelucca/pixel-agents) — это расширение для **VS Code** и завязано на **Claude Code CLI** (персонажи в виртуальном офисе). В Cursor свои агенты и другая среда; расширение из маркетплейса VS Code в Cursor может не подойти или не быть доступно.  
  Итого: если нужна именно визуализация «офис с персонажами» — используют отдельно **VS Code + Claude Code + Pixel Agents**. Если достаточно работы агентов в Cursor без пиксель-офиса — всё поднимается из Cursor, без VS Code.

### 3. Сводная картинка потока

```
[Вы]  →  Отдельный интерфейс (телефон/браузер)  →  Очередь задач для ИИ
                «Поставить задачу», выбор роли/скилла
                                    ↓
[Cursor]  ←  Агент по скиллу запрашивает очередь  ←  API очереди
                Берёт задачу, выполняет, обновляет статус
```

Либо, при желании визуализации:

```
[Вы]  →  Отдельный интерфейс  →  Очередь для ИИ
                                    ↓
[VS Code + Pixel Agents + Claude Code]  ←  Агент забирает задачу из API
                Персонажи в офисе, активность на экране
```

## Что нужно сделать в проекте

1. **Отдельный интерфейс для ИИ-команды** — не Flow Universe: свой UI (страница или отдельное приложение) + бэкенд/API очереди задач для агентов (хранение, статусы, привязка к ролям/скиллам). Flow Universe для этого не использовать.
2. **Скилл для агентов в Cursor** — «проверять очередь задач для ИИ» и «брать задачу в работу» обращаются к этому новому API, а не к Flow Universe.
3. **Cursor как основная среда** — постановка задач в отдельном интерфейсе, агенты в Cursor; VS Code и Pixel Agents — опционально, только если нужна визуализация офиса. **Установка VS Code и Pixel Agents:** пошагово в **docs/SETUP_VSCODE_PIXEL_AGENTS.md**.

После появления отдельного интерфейса и API документ и скилл можно обновить под конкретные URL и форматы (например переименовать/заменить `tasktime-inbox` на `ai-queue` и т.п.).
