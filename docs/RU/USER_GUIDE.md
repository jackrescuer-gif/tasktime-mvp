# Инструкция пользователя TaskTime

Краткое руководство по работе с веб-интерфейсом TaskTime: задачи, учёт времени, отчёты.

---

## Вход в систему

1. Откройте адрес приложения в браузере (Chrome или Yandex Browser рекомендуется).
2. Введите **логин** (email) и **пароль**.
3. Нажмите **Войти**.

Логин — это адрес электронной почты в латинице. Пароль задаётся администратором или при регистрации (если включена).

---

## Выход и смена учётной записи

- В правом верхнем углу нажмите **Выйти**. Токен сессии будет удалён, откроется экран входа.
- Чтобы войти под другим пользователем, после выхода введите другой логин и пароль.

---

## Задачи

### Список задач

- На главной странице отображаются задачи (по умолчанию — все, с учётом ваших прав: вы видите задачи, где вы автор или исполнитель, либо все задачи, если вы администратор/менеджер).
- **Фильтры:** «Все», «В работе», «Готово», «Мои» — позволяют сузить список.
- **Вид:** переключатель **Список** / **Таблица** — карточки или таблица с колонками (Название, Статус, Исполнитель, Приоритет, Действия).
- **Ширина столбцов (таблица):** в табличном виде можно изменить ширину колонки, потянув за правый край заголовка. Ширина сохраняется в браузере и восстанавливается при следующем входе.

### Создание задачи

1. Нажмите **Новая задача**.
2. Заполните название (обязательно), при необходимости — описание, исполнителя, приоритет.
3. Нажмите **Создать**.

### Редактирование и статусы

- **Начать работу** — запускает таймер по задаче и переводит её в статус «В работе».
- **Стоп** — останавливает таймер и фиксирует затраченное время.
- Клик по чекбоксу (или кнопка «Готово» в таблице) переводит задачу в «Готово» или обратно в «Открыта».

Типы статусов: **Открыта**, **В работе**, **Готово**. Приоритеты: Низкий, Средний, Высокий.

---

## Учёт времени

- Таймер запускается по кнопке **Начать работу** / **Старт** у задачи. В шапке отображается текущий счётчик и кнопка **Стоп**.
- Остановка таймера сохраняет запись о затраченном времени по задаче.
- Данные используются в разделе **Отчёт по времени**.

---

## Отчёт по времени

- Вкладка **Отчёт по времени** в навигации.
- **По дням** — сумма времени по каждому дню.
- **По задачам** — сумма времени по каждой задаче.
- В таблицах отчёта можно менять ширину столбцов (потянуть за край заголовка); ширина сохраняется в браузере.

---

## Роли и доступ

- **user** — видит и может редактировать только задачи, где вы автор (creator) или исполнитель (assignee).
- **manager** / **admin** — видят и могут редактировать все задачи.

Если какое-то действие недоступно (например, просмотр чужой задачи), приложение вернёт ошибку «Forbidden».

## AI-функции (Sprint 4)

### AI-оценка трудоёмкости

Позволяет автоматически оценить, сколько часов займёт выполнение задачи.

1. Откройте карточку любой задачи (TASK, STORY, BUG, EPIC, SUBTASK).
2. В правой панели найдите блок **AI Estimate**.
3. Нажмите кнопку **AI Estimate**.
4. Подождите несколько секунд — AI проанализирует название и описание задачи.
5. Результат:
   - **Часы** — оценочное количество часов.
   - **Уверенность** (confidence): `high` — описание детальное; `medium` — достаточно данных; `low` — мало информации.
   - **Обоснование** — краткое объяснение оценки.
6. Оценка автоматически сохраняется в поле «Estimated hours» задачи.

> Совет: чем подробнее описание задачи, тем точнее оценка. Добавьте критерии приёмки, список шагов — confidence станет `high`.

### AI-декомпозиция задач

Доступна только для EPIC и STORY. Помогает разбить крупную задачу на конкретные подзадачи.

1. Откройте карточку EPIC или STORY.
2. В правой панели найдите блок **AI Decompose**.
3. Нажмите кнопку **Decompose into STORYs** (для EPIC) или **Decompose into TASKs** (для STORY).
4. AI предложит список дочерних задач (обычно 3–7 штук) с описанием и оценкой часов.
5. Все задачи выбраны по умолчанию. Снимите галочки с ненужных.
6. Нажмите **Create selected (N)** — задачи будут созданы в проекте.
7. Обновите страницу или раздел «Sub-issues» — новые задачи появятся в иерархии.

## Telegram-нотификации

Получайте уведомления о событиях в задачах прямо в Telegram.

### Подключение

1. Перейдите в раздел **Admin** → блок **Telegram Notifications**.
2. Найдите и запустите бота **@tasktime_notify_bot** в Telegram (или тот бот, который настроил ваш администратор).
3. Бот пришлёт сообщение с вашим **chat ID** (числовой идентификатор).
4. Скопируйте chat ID и вставьте в поле «Your Telegram chat ID».
5. Нажмите **Connect**.

После подключения вы будете получать уведомления когда:
- На вас назначена задача.
- Статус вашей задачи изменился.
- Кто-то оставил комментарий к вашей задаче.

### Отключение

В блоке **Telegram Notifications** нажмите **Disconnect** и подтвердите.

## Экспорт отчётов

Менеджеры и администраторы могут выгружать данные в файлы.

1. Перейдите в раздел **Admin** → блок **Reports**.
2. Выберите **Проект** (обязательно), при необходимости — Спринт и Период.
3. Нажмите одну из кнопок:
   - **Issues CSV** — таблица задач в формате CSV (открывается в Excel/LibreOffice).
   - **Issues PDF** — таблица задач в формате PDF.
   - **Time CSV** — журнал списания времени в формате CSV.
4. Файл скачается автоматически.

> CSV-файлы используют кодировку UTF-8. Если Excel отображает кириллицу некорректно, при открытии выберите кодировку UTF-8 в мастере импорта.

## GitLab-интеграция (для тимлидов)

Статусы задач обновляются автоматически при событиях в GitLab:

| Событие GitLab | Статус TaskTime |
|---------------|----------------|
| Push коммита с ключом задачи | IN_PROGRESS |
| Открыт Merge Request | IN_PROGRESS |
| MR влит (merged) | REVIEW |
| MR закрыт | DONE |

Чтобы задача обновилась, упомяните её ключ (например `PROJ-42`) в названии коммита или MR.

Настройку интеграции выполняет администратор в разделе **Admin → GitLab Integration**.
