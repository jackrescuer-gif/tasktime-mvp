## Область проекта «MVP замены Jira на вайб-коде»

### Назначение

Эта область проекта в Flow Universe описывает реальный рабочий проект «MVP замены Jira на вайб-коде».  
Цель — собрать, приоритизировать и реализовать задачи по созданию компактного, «вайб»-ориентированного таск‑трекера, который сможет заменить Jira для внутренней команды.

Проект живой: в нём будут появляться новые эпики, сториз и сабтаски, отражающие реальные шаги по разработке, тестированию, деплою и внедрению MVP.

### Типы проектов

- `real` — живые продуктовые проекты (в том числе «MVP замены Jira на вайб-коде»).  
- `demo` — синтетические/демо‑проекты, используемые как демонстрационный материал и сценарии.

В БД это отражается полем `projects.project_type` (`demo` по умолчанию, `real` для боевых / живых проектов).

### Use-cases для области проекта

1. **Добавление задач в реальный проект**
   - Пользователь или ИИ‑агент создаёт эпики/сториз/сабтаски с `context_type = 'project'` и `context_id`, указывающим на проект «MVP замены Jira на вайб-коде`.
   - Новые задачи по умолчанию попадают в колонку Kanban **Backlog** (`status = 'open'`).

2. **Просмотр задач в Kanban‑режиме**
   - Для проекта «MVP замены Jira на вайб-коде» в UI доступен Kanban‑борд с колонками:
     - Backlog (`status = 'open'`)
     - In progress (`status = 'in_progress'`)
     - Done (`status = 'done'`)
   - В колонках отображаются, как минимум, сториз этого проекта (уровень `story`), при необходимости — отдельные карточки для сабтасков.

3. **Перемещение задач между колонками**
   - Пользователь меняет статус задачи из UI (кнопками на карточке).
   - На бэкенде используется `PATCH /api/task-items/:id/status`, при этом:
     - переход в **In progress** ставит `status = 'in_progress'`;
     - переход в **Done** ставит `status = 'done'`;
     - возврат в **Backlog** ставит `status = 'open'`.

4. **Фильтрация реального проекта от демо**
   - На странице «Проекты» в UI:
     - реальный проект «MVP замены Jira на вайб-коде» отображается в отдельном блоке «Реальные проекты»;
     - демо‑проекты остаются в секции «Демо‑проекты» и визуально приглушены;
   - В ответе `GET /api/projects` фронтенд использует `project_type`, чтобы правильно разделить проекты.

5. **Совместная работа людей и агентов**
   - Эпики/сториз в этом проекте могут создаваться и редактироваться как людьми, так и ИИ‑агентами.
   - В бизнес‑контексте «MVP замены Jira…» используется как единый бэклог для дальнейшей реальной разработки.

### Связь с существующими сущностями Flow Universe

- Пользователь (`users`) — владелец задач и проекта (owner_id).
- Проект (`projects`) — живой или демо‑контейнер задач, отличающийся полем `project_type`.
- Иерархические задачи (`task_items`) — эпики/сториз/сабтаски с `context_type = 'project'` и `context_id = projects.id`.
