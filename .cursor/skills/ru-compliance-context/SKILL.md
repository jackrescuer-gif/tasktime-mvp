# Контекст импортозамещения и норм (РФ)

Все агенты должны учитывать этот контекст при решениях в своих ролях. Юридические выводы — за юристом или ответственным по комплаенсу; скилл задаёт структуру требований и допущения.

## Важно про систему

**Система не является объектом КИИ** (не подпадает под 187-ФЗ «О безопасности критической информационной инфраструктуры»). К ней не применяются режимы и ограничения, обязательные для объектов КИИ (перечни, категорирование, запреты на иностранное ПО по правилам для КИИ и т.п.).

## Структура требований по импортозамещению в РФ

1. **Законодательство** — федеральные законы (в т.ч. 187-ФЗ для КИИ), указы.
2. **Подзаконные акты** — постановления Правительства (напр. № 1236, 1478, 1875, 1912, 658), реестр российского ПО.
3. **Отраслевые и ведомственные нормы** — отраслевые стандарты, методички (напр. Минцифры).
4. **Политики заказчика/организации** — внутренние правила, контракты, требования к поставщикам.

Для системы, не являющейся ЗоКИИ, жёсткие запреты по иностранному ПО (как для госорганов и КИИ) не действуют автоматически. Имеют значение отрасль, контракт и политика заказчика.

## Допущения и легальные варианты

- **Не КИИ** — допускается рассматривать использование иностранного ПО и оборудования в рамках: контракта, политики компании, отсутствия обработки данных КИИ/госсекретов.
- **Переходные периоды и исключения** — нормы могут предусматривать сроки и исключения при отсутствии российских аналогов или по согласованию; для не-КИИ набор ограничений уже.
- **Критерии «российского» ПО** — реестр Минцифры, структура владения, место разработки; при выборе решений учитывать, но не подменять юридическую оценку.

Агент не должен по умолчанию предполагать необходимость полного импортозамещения или автоматического применения норм ЗоКИИ. При сомнениях — формулировать допущения и рекомендовать согласование с юристом/комплаенсом.
