# Входящие задачи для ИИ-команды

## Назначение

Очередь задач **для ИИ-агентов** (не для пользователей TaskTime). Пользователь ставит задачи в **отдельном интерфейсе** для ИИ-команды (с телефона или из браузера). Агент проверяет эту очередь и может взять задачу в работу.

**Важно:** TaskTime — система для людей (задачи пользователей, учёт времени). Для постановки задач ИИ-команде и работы со скиллами нужен **отдельный интерфейс и API** (в разработке). Когда они появятся, скилл будет обращаться к ним.

## Временный вариант (пока нет отдельной очереди)

Если в проекте настроена очередь через TaskTime API (отдельный бот-пользователь, только для теста):

- `TASKTIME_URL` — базовый URL, `TASKTIME_TOKEN` — JWT бота.
- Запрос: `GET {TASKTIME_URL}/api/tasks?status=open`, заголовок `Authorization: Bearer {TASKTIME_TOKEN}`.
- После выполнения: `PUT {TASKTIME_URL}/api/tasks/:id` с `status: 'done'` или `in_progress`.

Если переменные не заданы или очередь для ИИ вынесена в отдельный сервис — агент сообщает, что нужно настроить очередь для ИИ (см. docs/PIXEL_AGENTS_AND_TASKS.md).

## Поведение агента

1. По запросу («что в очереди?», «возьми задачу») или при старте — при наличии настроенной очереди запросить открытые задачи.
2. Показать список (название, приоритет, роль/скилл при наличии), предложить взять в работу или уточнить у пользователя.
3. После выполнения обновить статус задачи в очереди (по текущему API).

## Связь с интерфейсом и визуализацией

- **Постановка задач ИИ-команде:** отдельный интерфейс (не TaskTime), описание в **docs/PIXEL_AGENTS_AND_TASKS.md**.
- **Где работают агенты:** в первую очередь **Cursor** (без отдельного VS Code). Визуализация пиксель-офиса — расширение [Pixel Agents](https://github.com/pablodelucca/pixel-agents) для VS Code + Claude Code, опционально.
