# Роли агентов и скиллы

## Общий контекст (обязателен для всех агентов)

**Импортозамещение и нормы РФ.** Все агенты должны учитывать скилл **`ru-compliance-context`**:

- Структура требований по импортозамещению в РФ (законы, подзаконные акты, отрасль, политики заказчика).
- **Система не является объектом ЗоКИИ** (не 187-ФЗ) — к ней не применяются режимы и ограничения КИИ.
- Допущения и легальные способы использования иностранного ПО и оборудования; не предполагать по умолчанию полное импортозамещение или нормы КИИ.
- Итоговые юридические выводы — за юристом или ответственным по комплаенсу.

Скилл: `.cursor/skills/ru-compliance-context/SKILL.md`

## Обязательный пайплайн разработки

Любая задача с изменением кода, API, БД или инфраструктуры проходит этапы **строго по порядку**:

```
[1] GATEKEEPER         MCP validate_and_improve(request) → архитектурное заключение
       ↓
[2] АНАЛИТИКА          system-analyst → требования, use-cases, API-контракт
       ↓
[3] ИБ                 infosec → риски, auth, роли, данные
       ↓
[4] ТЕСТ-ПЛАН          tester → unit, QA авто+ручные, UAT (до разработки)
       ↓
[5] РАЗРАБОТКА         developer → реализация по спецификации
       ↓
[6] ТЕСТИРОВАНИЕ       tester → прогон тест-плана, все тесты зелёные
       ↓
[7] ПРОВЕРКА           → показать Павлу, получить явное «ок»
       ↓
[8] ДЕПЛОЙ             push → GitHub Actions → /var/log/tasktime-deploy.log
```

**Деплой без явного подтверждения пользователя — запрещён.**  
Полный скилл: `.cursor/skills/tasktime-workflow/SKILL.md`

## Роли и скиллы

| Роль | Скилл | Когда привлекать |
|------|--------|-------------------|
| **Корпоративный архитектор** | `corporate-architect` | Встраивание бизнес-функции в корп. ИТ-ландшафт, границы, интеграции, архитектурные решения. |
| **Разработчик** | `developer` | Реализация фич, рефакторинг, выбор технологий и библиотек. |
| **Тестировщик (авто + ручные)** | `tester` | Автотесты (unit, API, e2e), ручные сценарии, приёмочное тестирование. |
| **Системный аналитик** | `system-analyst` | Требования, use cases, спецификации, документация для нетехнарей. |
| **Специалист по информ. безопасности** | `infosec` | ИБ-меры, разграничение доступа, защита данных, комплаенс по ИБ. |
| **Специалист по сети и инфраструктуре** | `network-infrastructure` | Сети, серверы, облако, деплой, доступность. |
| **Deploy TaskTime** | `deploy-tasktime` | Деплой приложения на Ubuntu/Timeweb (частный кейс инфраструктуры). |
| **Документация TaskTime** | `docs-tasktime` | Отчёты «что сделано», формат учёток, DEPLOY / ACCOUNTS / DEPLOYMENT_STEPS. |
| **Процесс / workflow** | `tasktime-workflow` | Полный пайплайн: gatekeeper → аналитика → ИБ → тест-план → разработка → тестирование → проверка с Павлом → деплой. |
| **TaskTime — входящие задачи** | `tasktime-inbox` | Проверка очереди задач (GET /api/tasks), приоритизация, обновление статуса. Для удалённой постановки задач с телефона — веб-интерфейс TaskTime. |

## Интерфейс для ИИ-команды и визуализация

- **TaskTime** — только для **пользователей** (люди, задачи, учёт времени). Для ИИ-команды не используется.
- **Постановка задач ИИ-команде:** нужен **отдельный интерфейс** (очередь задач для агентов, выбор роли/скилла, с телефона или из браузера). В проекте пока не реализован — см. **docs/PIXEL_AGENTS_AND_TASKS.md** и техдолг.
- **Где работают агенты:** в первую очередь **Cursor** (всё можно поднять из Cursor, VS Code не обязателен). Визуализация «офис с персонажами» — расширение [Pixel Agents](https://github.com/pablodelucca/pixel-agents) для VS Code + Claude Code; в Cursor может быть недоступно. Подробно: **docs/PIXEL_AGENTS_AND_TASKS.md**.

## Расположение скиллов

Все скиллы в `.cursor/skills/`:

| Скилл | Назначение |
|--------|------------|
| `ru-compliance-context` | Общий контекст импортозамещения и норм (не ЗоКИИ); подключать ко всем ролям. |
| `corporate-architect` | Корпоративный архитектор. |
| `developer` | Разработчик. |
| `tester` | Тестировщик (авто + ручные). |
| `system-analyst` | Системный аналитик. |
| `infosec` | Специалист по информ. безопасности. |
| `network-infrastructure` | Сеть и инфраструктура. |
| `deploy-tasktime` | Деплой TaskTime на VPS. |
| `docs-tasktime` | Документация по деплою и учёткам. |
| `tasktime-workflow` | Полный пайплайн: gatekeeper → аналитика → ИБ → тест-план → разработка → тестирование → проверка → деплой. |
| `tasktime-inbox` | Очередь задач TaskTime для ИИ-команды; проверка открытых задач, обновление статусов. |

При вызове агента указывать роль или скилл; для решений по архитектуре, разработке, ИБ, инфраструктуре — подключать **`ru-compliance-context`** вместе с ролью.
