# Техдолг

## 1. Привести авторизацию к околобоевому формату

Сейчас: упрощённая схема — логин = пароль, фиксированный список user1…user10, проверка только на клиенте.

Нужно:
- [ ] Реализовать авторизацию через бэкенд (сессии или JWT)
- [ ] Хранить пользователей и хэши паролей в БД (или интеграция с существующей схемой users)
- [ ] Убрать валидацию «только user1–user10» на клиенте и заменить на запрос к API логина
- [ ] Защитить API от неавторизованных запросов (проверка токена/сессии)

## 2. Добавить скиллы по банковской и депозитарной тайне РФ

- [ ] Скилл по банковской тайне (нормы РФ, объём, исключения, последствия разглашения).
- [ ] Скилл по депозитарной тайне (нормы РФ, связь с учётом ценных бумаг, ограничения).
- [ ] Подключить контекст к ролям, работающим с персональными/финансовыми данными (аналитик, ИБ, архитектор).

## 3. Отдельный интерфейс и API для ИИ-команды

TaskTime — для пользователей (люди). Для постановки задач ИИ-агентам и работы со скиллами нужен отдельный контур:

- [ ] Отдельный интерфейс (страница или приложение): постановка задач для агентов, выбор роли/скилла, очередь (открытые / в работе / выполненные). Доступ с телефона и из браузера.
- [ ] API очереди задач для ИИ: создание задачи, список с фильтрами (роль, статус), обновление статуса агентом.
- [ ] Скилл для агентов в Cursor: запрос очереди из этого API, «взять задачу», обновить статус (заменить или дополнить tasktime-inbox).
